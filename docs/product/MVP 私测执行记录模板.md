# MVP 私测执行记录模板

> 本文档基于《21天总准则》《新MVP PRD》《MVP PM决策共识》《MVP UX准则》和《MVP 测试用例与私测观察标准》整理。
> 当前文档用于第一轮私测过程记录，不进入代码实现、数据埋点实现或正式运营分析。

## 1. 使用目的

第一轮私测不是验证商业化，也不是验证功能完整度，而是验证：

```text
行动 + 记录 + 轻复盘闭环是否成立。
```

本模板用于记录：

- 用户第一次是否能拿到今日行动卡。
- 用户是否理解今天只做什么。
- 用户是否真的完成行动。
- 用户是否愿意留下真实记录。
- 用户是否第 2-3 天回来。
- 轻复盘是否能给出具体下一步。
- 7 天内是否完成 2 次真实推进。

不记录或不重点分析：

- 商业化收入。
- 页面访问量。
- 功能模块数量。
- AI 输出字数。
- 报告长度。
- 是否完整测满 21 天。

## 6. 未完成行动分析记录

当用户没有完成行动时填写。

| 项目 | 记录 |
|---|---|
| 用户编号 |  |
| 未完成的行动 |  |
| 用户自己说的原因 |  |
| 判断属于哪类阻力 | 太大 / 太难 / 太模糊 / 缺材料 / 缺时间 / 不信任 / 其他 |
| 是否需要缩小行动 | 是 / 否 |
| 缩小后的行动 |  |
| 用户是否愿意继续 | 是 / 否 |

产品处理原则：

- 不评价用户。
- 不制造失败感。
- 不强行进入下一步。
- 把行动调回伸展区。

推荐记录表达：

```text
用户没有完成，不一定是意愿问题，先判断行动是否过大、过难或过模糊。
```

| 复盘日期 |  |
| 复盘依据是什么 |  |
| 看到的线索是否来自真实记录 | 是 / 否 |
| 是否指出信息缺口 | 是 / 否 |
| 是否给出 1 个下一步行动 | 是 / 否 |
| 是否有泛泛建议感 | 是 / 否 |
| 是否有评价用户感 | 是 / 否 |
| 用户是否觉得复盘具体 | 是 / 否 |
| 用户是否愿意按下一步继续 | 是 / 否 |

一票否决：

- 没有真实记录却生成深度复盘。
- 把无反馈归因为用户本人不行。
- 编造岗位要求、公司反馈或筛选规则。
- 输出匹配度、录取概率或适合度。

高严重度问题包括：

- 显示 API key、prompt、token、模型名、错误堆栈或调试信息。
- 编造经历、JD、项目结果或投递反馈。
- 输出匹配度、录取概率或适合度。
- 判断用户本人不适合或能力不行。
- 泄露用户真实经历、JD、投递记录、反馈记录或简历片段到无关页面。
